An Application Keeps Crashing

Use this guide to diagnose and resolve an application that closes unexpectedly, freezes, or produces errors repeatedly on your Robotix IT laptop.

Start Here

Identify the specifics: - Which application is crashing? (e.g., Microsoft Outlook, Teams, a line-of-business app) - What happens exactly? (crashes immediately on open, crashes after a specific action, freezes and becomes unresponsive, shows an error code) - When did it start? (after an update, after a specific action, or seemingly at random) - Is this a Robotix IT-managed application? (pre-installed or installed via Company Portal / Self Service) or a third-party app?

Write down any error codes or pop-up messages word-for-word — they are critical for troubleshooting.

Step-by-Step Diagnostic Flow

1. Restart the application.

Force-quit the application fully and reopen it.

Windows: Press Ctrl + Shift + Esc to open Task Manager. Find the application in the list, right-click it, and select End Task. Wait 15 seconds and reopen the app.

Mac: Press Cmd + Option + Esc to open Force Quit Applications. Select the app and click Force Quit. Wait 15 seconds and reopen.

Did restarting the application resolve the issue?

If yes (it works now but has crashed before) → Note the crash and monitor. If it happens more than twice in a week, submit a ticket at https://help.robotix.com with the pattern details.

If no (it crashes again immediately or shortly after) → Proceed to Step 2.

2. Restart your laptop.

Many application issues are resolved by a full restart, which clears memory, applies pending updates, and resets background processes.

Windows: Use Start > Power > Restart (not Shut Down). Do not use Shut Down followed by Power On if Fast Startup is enabled.

Mac: Use Apple menu > Restart.

Did restarting the laptop resolve the crash?

If yes → Monitor for recurrence. If it happens again after a clean restart, proceed to Step 3.

If no → Proceed to Step 3.

3. Is there a pending software update for the application?

Crashes are often caused by bugs in older versions. Check for updates:

Windows (Microsoft 365 apps): Open any Office app (Word, Outlook), go to File > Account > Update Options > Update Now.

Mac (Microsoft 365 apps): Open Microsoft AutoUpdate (found in Applications > Microsoft AutoUpdate or launched from any Office app under Help > Check for Updates).

Company Portal (Windows) / Self Service (Mac): Open the portal and check the Updates tab for available patches.

Did installing updates resolve the crash?

If yes → The app was running an unstable version. Monitor after the update.

If no → Proceed to Step 4.

4. Is the crash specific to one particular file, action, or window?

If yes (e.g., Outlook crashes when opening a specific email, Teams crashes in a specific meeting room) → - For Outlook: Try opening Outlook in Safe Mode (Windows: hold Ctrl while launching Outlook, then click Yes). If Outlook is stable in Safe Mode, a corrupt add-in is the likely cause — disable add-ins one by one under File > Options > Add-ins. - For a specific file: try opening a different file of the same type to isolate whether the file or the app is corrupt. - Note the exact action that causes the crash and include it in your ticket.

If no (crashes randomly regardless of what you are doing) → Proceed to Step 5.

5. Does the issue occur with a freshly created user profile or guest account?

Windows: Go to Settings > Accounts > Other users, add a temporary local account, and test the application under that account.

Mac: Go to System Settings > Users & Groups, create a temporary user, log in as that user, and test the app.

If the app is stable in the new profile → Your user profile is likely corrupted. Submit a ticket at https://help.robotix.com (P3 – Medium) — IT can migrate your profile.

If the app crashes in the new profile too → The issue is system-wide, not user-specific. Proceed to Step 6.

6. Check the system event log for crash details.

Windows: Open Event Viewer (search in Start menu), navigate to Windows Logs > Application, and filter for Error level events. Look for entries with the application name around the time it crashed. Screenshot or copy the error code and description.

Mac: Open Console (Applications > Utilities > Console) and look under Crash Reports for entries matching the application name. Note the exception type and crashing thread.

Does the log show a specific error code or module name?

If yes → Include this in your support ticket — it dramatically speeds up diagnosis.

If no meaningful error found → Proceed to Step 7.

7. Repair or reinstall the application.

Microsoft 365 on Windows: 1. Go to Settings > Apps > Installed Apps, search for "Microsoft 365." 2. Click the three-dot menu and select Modify. 3. Choose Quick Repair first; if that fails, run Online Repair.

Other apps (Windows): Uninstall via Settings > Apps and reinstall from Company Portal.

Mac apps: Delete the application from Applications, then reinstall from Self Service. Also remove any associated preference files in ~/Library/Preferences/ if recommended by the Service Desk.

Did repairing/reinstalling resolve the crash?

If yes → Monitor for 24 hours. If the issue returns, contact the Service Desk.

If no → Proceed to Step 8.

8. Does the issue affect other users on the same team or with the same application?

Ask a colleague if they experience the same crash with the same application.

If yes (multiple users affected) → This may be a platform-wide or update-related issue. Submit a ticket at https://help.robotix.com and note that multiple users are affected — this escalates the priority.

If no (only you) → The issue is isolated to your device or account. Contact the Service Desk for a hands-on investigation.

Quick Checklist

Before calling the Service Desk, confirm you have tried:

  • [ ] Force-quit and relaunched the application
  • [ ] Restarted the laptop
  • [ ] Checked for and installed all pending updates for the application
  • [ ] Tested whether a specific file or action triggers the crash
  • [ ] Tried running the app in Safe Mode (Outlook/Windows) or under a new user profile
  • [ ] Checked Event Viewer (Windows) or Console/Crash Reports (Mac) for error codes
  • [ ] Attempted a Quick Repair or reinstall via Company Portal / Self Service
  • [ ] Confirmed whether colleagues with the same app are experiencing the same issue
