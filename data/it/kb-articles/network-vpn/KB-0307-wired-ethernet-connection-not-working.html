Wired / Ethernet Connection Isn't Working

Summary

A wired Ethernet connection should be faster and more reliable than Wi-Fi. If your wired connection is not working — no internet, no link light on the cable, or the connection is not recognized — the cause is usually a loose cable, a dock/adapter issue, or a disabled network adapter. This article steps through diagnosing and fixing wired connectivity problems on both Windows 11 and macOS. Robotix IT standard docking hardware is the Dell WD19 and USB-C docks.

Before You Begin

  • Identify how your device is connected: directly via an Ethernet cable plugged into the laptop, through a Dell WD19 dock, or through a USB-C to Ethernet adapter.
  • Check that you can see a link light (usually amber/green) on the Ethernet port on the dock or wall jack. No light usually means a physical layer problem.
  • Have your Okta credentials available in case you need to reconnect to Wi-Fi as a temporary workaround.

Steps

Step 1 — Check the physical connections

Before touching software, inspect the hardware:

  1. Unplug and firmly re-seat the Ethernet cable at both ends — at the dock/adapter and at the wall jack or switch.
  2. Check for a link light on the Ethernet port: - Solid amber or green = link established. - No light = no physical connection detected.
  3. If using a Dell WD19 dock, ensure the power brick is connected and the dock's power LED is on. Without power, the dock's network adapter does not function.
  4. Try a different Ethernet cable if available. Cables can fail, especially if kinked or damaged.
  5. Try a different wall jack at your desk or in a conference room to rule out a dead port.
  6. If you use a USB-C to Ethernet adapter, try a different USB-C port on your laptop.

Step 2 — Restart the dock and reconnect

  1. Unplug the USB-C or Thunderbolt cable from your laptop.
  2. Unplug the dock's power cable from the wall.
  3. Wait 30 seconds.
  4. Plug the power cable back in, then reconnect the dock to your laptop.
  5. Wait 20 seconds for the dock to initialize.
  6. Check the Ethernet link light again and test connectivity.

Step 3 — Check the network adapter in Windows 11

  1. Right-click the Start button and select Device Manager.
  2. Expand Network adapters.
  3. Look for an Ethernet adapter (often labeled "Realtek," "Intel," "DisplayLink," or "Aquantia" for dock adapters).
  4. If there is a yellow warning icon next to an adapter, right-click it > Update driver > Search automatically for drivers.
  5. If the adapter is greyed out (disabled), right-click > Enable device.
  6. Right-click the adapter again > Disable device, wait 5 seconds, then right-click > Enable device to force a reset.
  7. Open Settings > Network & internet > Ethernet and verify the connection now shows a status.

Step 4 — Check the network adapter on macOS

  1. Open System Settings > Network.
  2. Look for a Thunderbolt Ethernet, USB LAN, or DisplayLink adapter in the left pane. - If it shows with a yellow or red dot, the interface is connected but not getting an IP address. - If the adapter is not listed at all, macOS has not detected the hardware — revisit Step 1 and Step 2.
  3. Click the adapter, then click Details.
  4. Under the TCP/IP tab, click Renew DHCP Lease.
  5. If the IP address remains 169.254.x.x (APIPA), the device is not reaching the DHCP server — see Step 6.

Step 5 — Release and renew the IP address (Windows)

An APIPA address (169.254.x.x) means your PC obtained no IP from the network's DHCP server.

  1. Open Windows Terminal or Command Prompt as Administrator.
  2. Run: ipconfig /release ipconfig /flushdns ipconfig /renew
  3. Run ipconfig /all and verify the IPv4 Address is now a valid office address (e.g., 10.x.x.x or 172.16.x.x), not 169.254.x.x.
  4. Test connectivity by opening a browser.

Step 6 — Reset the network stack (Windows)

If renewing the IP did not help, reset the TCP/IP and Winsock stack:

  1. Open Windows Terminal as Administrator.
  2. Run each command in sequence: netsh winsock reset netsh int ip reset ipconfig /flushdns
  3. Restart your computer — these changes require a reboot.
  4. Reconnect the dock after the restart and test.

Step 7 — Try Wi-Fi as a temporary workaround

If you need to keep working while troubleshooting:

  1. Connect to Robotix-Corp Wi-Fi (see KB-0300).
  2. Leave the faulty Ethernet connected — Windows and macOS will typically prefer the wired adapter if it comes back online.

Step 8 — Report a suspected wall jack or switch port issue

If you have tried all steps above, a different cable, and a different port — and link lights are still absent — the wall jack or office switch port may be faulty or not patched through to the network.

  1. Open a ticket at https://help.robotix.com, category: Network / Ethernet / Hardware.
  2. Include: - Your desk number and office location (e.g., "Denver HQ, Desk 3-42B") - Asset tag of your laptop (SUM-LT-##### on the bottom of the device) - Steps already tried - Whether link lights are present
  3. An L3 network technician will schedule a site visit.

Troubleshooting Quick Reference

Symptom Likely Cause Fix
No link light on dock or cable Loose cable, bad cable, or dead port Re-seat cable; try different cable; try different wall jack
Link light on, but no internet DHCP not assigning IP Release/renew IP (Step 5); restart dock (Step 2)
IP shows 169.254.x.x DHCP failure Steps 5 and 6; contact Service Desk if persists
Adapter shows yellow warning in Device Manager Driver issue Update or reinstall dock driver (Step 3)
macOS doesn't show the adapter at all Dock not powered or USB-C issue Confirm dock power; try different USB-C port on Mac
Dock connected but only USB devices work, not network Dock driver not installed Install DisplayLink Manager from Jamf Self Service
Works after restart but fails within an hour Power management issue (Windows) Device Manager > Ethernet adapter > Properties > Power Management > uncheck "Allow computer to turn off this device to save power"
