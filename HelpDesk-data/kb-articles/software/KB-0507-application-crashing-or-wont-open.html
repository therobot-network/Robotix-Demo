An Application Keeps Crashing or Won't Open

Summary

Application crashes and launch failures are often caused by corrupted temporary files, pending software updates, conflicting processes, or insufficient disk space. This article walks you through a structured series of self-service steps to diagnose and resolve the most common causes before escalating to the Service Desk.

Before You Begin

  • Note the exact error message (if any) and the name and version of the application. You can find the version in Help > About (most Windows apps) or App Name > About (Mac menu bar).
  • Identify when the problem started. Did it begin after a recent update, a system restart, or after installing other software?
  • Check whether the problem occurs for other users on the same device — if a colleague with a different account can open the app, the issue is likely profile-specific.
  • Ensure your device has at least 10 GB of free disk space. Low disk space causes many unexpected application failures. Check via File Explorer > This PC (Windows) or Apple menu > About This Mac > More Info > Storage (Mac).

Steps

Step 1 — Force Quit and Restart the Application

Windows: 1. Press Ctrl + Shift + Esc to open Task Manager. 2. Find the application under the Processes tab. If the status shows Not responding, right-click it and select End Task. 3. Wait 30 seconds, then reopen the application from the Start menu.

macOS: 1. Press Command + Option + Esc to open Force Quit Applications. 2. Select the frozen application and click Force Quit. 3. Wait 30 seconds, then reopen the application from the Dock or Applications folder.

Step 2 — Restart Your Computer

A full restart clears memory leaks and applies any pending system updates that may be interfering with the application.

  1. Save all open work in other applications.
  2. Windows: Click Start > Power > Restart (not Shut Down).
  3. macOS: Click Apple menu > Restart.
  4. After the restart, open only the affected application and observe whether the problem persists.

Step 3 — Check for Application Updates

Windows (Company Portal): 1. Open Company Portal > Apps. 2. Click the Updates section. If an update is available for the affected application, click Update.

macOS (Self Service): 1. Open Self Service and look for the application in the list. An Update button will appear if a newer version has been published. 2. Also check the application's own Help menu for an Check for Updates option.

Step 4 — Clear the Application's Cache or Reset Its Settings

Many applications store temporary data that can become corrupted. Clearing this data forces the app to rebuild it fresh.

Microsoft 365 apps (Outlook, Word, Excel, etc.) — Windows: 1. Close all Office apps. 2. Press Windows + R, type %localappdata%\Microsoft\Office\16.0\OfficeFileCache and press Enter. 3. Delete all files in this folder (do not delete the folder itself). 4. Reopen the affected Office app.

Microsoft 365 apps — macOS: 1. Quit all Office apps. 2. Open Finder > Go > Go to Folder… and type ~/Library/Group Containers/UBF8T346G9.Office/. 3. Move the OfficeFileCache folder to the Trash. The app will recreate it on next launch.

Zoom — Windows and macOS: 1. In Zoom, click your profile picture > Settings > General > Clear Cache.

Slack — Windows and macOS: 1. In Slack, open the menu: Help > Troubleshooting > Clear Cache and Restart (Windows) or Slack > Troubleshooting > Clear Cache and Restart (Mac).

Step 5 — Reinstall the Application

If cache clearing did not help, reinstall the application to replace any damaged files.

  1. Uninstall the application first (see KB-0509 for safe uninstall steps).
  2. Open Company Portal (Windows) or Self Service (Mac).
  3. Find the application and click Install.
  4. Open the newly installed application and test for the issue.

Step 6 — Collect Diagnostic Information for the Service Desk

If none of the above steps resolved the issue, gather the following before contacting the Service Desk:

  • Application name and version number.
  • Exact error message (a screenshot is ideal).
  • Whether the crash happens immediately on launch or after a specific action.
  • Steps to reproduce the crash consistently.
  • Windows: Open Event Viewer (search in the Start menu), go to Windows Logs > Application, and look for error entries around the time of the crash. Note the Source and Event ID.
  • macOS: Open Console (Applications > Utilities), filter by the application name, and note any crash reports listed under Crash Reports in the left sidebar.

Troubleshooting

The app crashes only when opening a specific file. The file itself may be corrupted. Try opening a different file of the same type. If other files open successfully, the original file is the problem — recover it from OneDrive version history or a recent backup.

The app worked yesterday and nothing has changed. A background Intune or Jamf update may have applied a new policy or app version overnight. Restart your device and try again. Check Company Portal or Self Service for recent updates.

A Microsoft 365 app shows "Word has stopped working" immediately on Windows. Disable COM add-ins: open the app in safe mode by pressing and holding Ctrl while clicking the app icon. If it opens successfully, go to File > Options > Add-ins and disable add-ins one by one to find the culprit.

I get "Application can't be opened because it is from an unidentified developer" on macOS. This means the application was not installed through Self Service. Do not bypass this warning with ctrl-click Open — instead, install the approved version from Self Service or contact the Service Desk to have the application reviewed and approved.
