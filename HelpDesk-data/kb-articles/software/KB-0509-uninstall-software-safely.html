Uninstall Software Safely

Summary

Removing software from a Summit IT-managed device should be done through the correct channels to ensure device compliance is maintained, licensing is returned where applicable, and no leftover files cause future issues. This article covers how to uninstall IT-managed applications through Company Portal (Windows) and Self Service (Mac), as well as how to remove non-catalog software that may have been installed with temporary administrator rights.

Before You Begin

  • Some applications cannot be uninstalled by end users — they are protected by Intune or Jamf policy because they are required for device security (e.g., CrowdStrike Falcon, Okta Verify, GlobalProtect). Attempts to remove these apps will fail silently or be reinstalled automatically. If you believe a protected app is causing a problem, contact the Service Desk rather than trying to remove it.
  • Uninstalling Microsoft 365 applications (Word, Outlook, etc.) will remove access to your email and files in those apps. Only uninstall Microsoft 365 apps if instructed by the Service Desk or if you plan to reinstall immediately.
  • Uninstalling an application does not automatically return a software license. For paid software (e.g., Adobe Acrobat Pro), notify the Service Desk after uninstalling so the license seat can be reassigned.
  • You do not need local administrator rights to uninstall software that was installed via Company Portal or Self Service.

Steps

Windows — Uninstall via Company Portal (Recommended)

Company Portal is the preferred method for removing IT-managed applications because it ensures the device compliance record is updated correctly.

  1. Open Company Portal from the Start menu.
  2. Click Apps in the left navigation pane.
  3. Find the application you want to remove. You can filter by Installed Apps using the dropdown at the top of the app list.
  4. Click the application tile to open its detail page.
  5. Click Uninstall. Confirm when prompted. A progress indicator will appear; do not close Company Portal until the status changes to Not Installed.

Windows — Uninstall via Windows Settings (Non-Catalog Apps)

For applications not managed through Company Portal (e.g., installed with temporary admin rights):

  1. Click the Start menu and go to Settings > Apps > Installed apps.
  2. Scroll to or search for the application you want to remove.
  3. Click the three-dot menu (···) to the right of the app name and click Uninstall.
  4. Follow the uninstaller prompts to completion. Some applications require a restart to finish removal.
  5. After uninstalling, notify the Service Desk via https://help.summitit.com if the application was installed under a temporary admin grant, so the removal can be logged.

macOS — Uninstall via Self Service (Recommended)

Some IT-managed applications published in Self Service have a corresponding uninstall policy.

  1. Open Self Service from your Applications folder or Dock.
  2. Search for the application using the search bar.
  3. If an Uninstall button is visible alongside or in place of Install, click it. The application will be removed silently.
  4. Note: Not all catalog apps have a Self Service uninstall policy. If no Uninstall button appears, use the Finder method below.

macOS — Uninstall via Finder (Non-Catalog Apps and Most Mac Apps)

  1. Open Finder and go to the Applications folder (click Applications in the Finder sidebar, or press Command + Shift + A).
  2. Locate the application you want to remove.
  3. Drag the application icon to the Trash, or right-click it and select Move to Trash.
  4. Empty the Trash: right-click the Trash icon in the Dock and select Empty Trash, or press Command + Shift + Delete in Finder.
  5. For thorough removal of associated preference files and caches, use the IT-approved app AppCleaner if it is available in Self Service. Do not use third-party "cleaner" apps not in the Self Service catalog.

Notify IT After Uninstalling Paid Software

  1. Go to https://help.summitit.com and sign in.
  2. Click Service Catalog > Software & Licensing > Release a Software License.
  3. Enter the application name, your device asset tag, and the date of uninstallation.
  4. Click Submit. The Service Desk will update the license inventory.

Troubleshooting

The Uninstall button is greyed out or missing in Company Portal. The application may be required by an IT policy and cannot be removed by end users. Contact the Service Desk to discuss why the application needs to be removed — IT can evaluate whether a policy exception is appropriate.

Windows Settings shows the app is uninstalled, but I can still see it in the Start menu. Restart your computer. The Start menu tile cache does not always update immediately after uninstallation. If the icon persists after a restart, right-click it and choose Unpin from Start. The underlying application has already been removed.

macOS: I dragged the app to Trash but it says "The item can't be moved to the Trash because it's open." Force quit the application first: press Command + Option + Esc, select the app, and click Force Quit. Then move it to Trash.

macOS: I uninstalled an app via Finder but its icon still appears in Launchpad. Click and hold the app icon in Launchpad until all icons start to wiggle, then click the X button on the icon to remove it from Launchpad. If no X appears, the app may have been reinstalled by a Jamf policy — contact the Service Desk.

I uninstalled a required application by mistake (e.g., CrowdStrike Falcon or GlobalProtect). Contact the Service Desk immediately. These applications protect your device and are required for network access. The Service Desk can push a reinstall remotely through Intune or Jamf, which typically takes 5–10 minutes.
