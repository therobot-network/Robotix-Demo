Wi-Fi or Internet Isn't Working

Use this guide to diagnose and resolve Wi-Fi and internet connectivity issues, whether you are in a Summit IT office or working remotely.

Start Here

Determine where you are and what you are connected to:

  • In a Summit IT office: You should be on the Summit-Corp Wi-Fi network (WPA2-Enterprise, Okta credentials) or wired Ethernet via your docking station.
  • At home or remote: You are on your personal internet connection, and should be connected to GlobalProtect VPN for internal resources.
  • Unknown/guest network: Confirm which Wi-Fi network your device shows as connected.

Check the Wi-Fi icon in your taskbar (Windows) or menu bar (Mac) to see your current connection status.

Step-by-Step Diagnostic Flow

1. Is Wi-Fi turned on on your laptop?

If no → Enable Wi-Fi via the taskbar (Windows: click the network icon, toggle Wi-Fi on) or menu bar (Mac: click the Wi-Fi icon, enable). Then attempt to connect to the correct network.

If yes → Proceed to Step 2.

2. Are you connected to a Wi-Fi network (shows a network name, not "Not Connected")?

If no → Proceed to Step 3 to connect to the correct network.

If yes → Proceed to Step 4 to verify the connection is working.

3. Connect to the correct Wi-Fi network.

In a Summit IT office: - Select Summit-Corp from the available networks list. - When prompted, enter your firstname.lastname@summitit.com username and Okta password. - If Summit-Corp does not appear in the list, proceed to Step 8.

At home or in a non-Summit location: - Connect to your home Wi-Fi using your home network password. - Once connected, launch GlobalProtect VPN (see FAQ-04 for VPN setup instructions).

Did connecting resolve the issue?

If yes → You are connected. No further action needed.

If no → Proceed to Step 4.

4. Can you load a basic website (e.g., https://google.com or https://help.summitit.com)?

If yes → Basic internet is working. If the issue is with a specific internal tool or resource, you may need to connect to GlobalProtect VPN. See Step 10.

If no → The connection is present but internet is not reaching your browser. Proceed to Step 5.

5. Is GlobalProtect VPN connected?

If yes and you cannot reach the internet → Try disconnecting from GlobalProtect. Sometimes VPN routing issues can block general internet access. Disconnect, test internet again without VPN, then reconnect VPN.

If no VPN and still no internet → Proceed to Step 6.

6. Flush DNS and renew your IP address.

Windows: 1. Open Command Prompt as Administrator. 2. Run: ipconfig /flushdns 3. Run: ipconfig /release then ipconfig /renew 4. Test internet again.

Mac: 1. Open Terminal. 2. Run: sudo killall -HUP mDNSResponder 3. Run: sudo ipconfig set en0 DHCP 4. Test internet again.

Did this resolve the issue?

If yes → Issue was a stale DNS or IP lease. Monitor for recurrence.

If no → Proceed to Step 7.

7. Restart your laptop's network adapter.

Windows: Go to Settings > Network & Internet > Advanced network settings > Network adapters, right-click your Wi-Fi adapter, and select Disable, then Enable after 10 seconds.

Mac: Go to System Settings > Wi-Fi, turn Wi-Fi off, wait 10 seconds, then turn it back on and reconnect.

Did this resolve the issue?

If yes → Adapter was in a bad state. Monitor for recurrence. If it happens often, submit a ticket for a persistent connectivity investigation.

If no → Proceed to Step 8.

8. Are other devices on the same network able to access the internet?

If yes (other devices work fine) → The issue is specific to your laptop. Restart your laptop and retry. If the issue persists after a restart, submit a ticket at https://help.summitit.com — there may be a driver or configuration issue.

If no (other devices also cannot connect) → The issue is with the network itself, not your laptop. Proceed to Step 9.

9. Is this in a Summit IT office?

If yes → This is a network infrastructure issue. Contact the Service Desk immediately at +1 (800) 555-7866 — if multiple users are affected, this may be a P1 or P2 incident. While waiting, use your mobile phone hotspot as a temporary workaround.

If no (home/remote network) → - Restart your home router and modem: unplug both from power, wait 30 seconds, plug the modem back in first, wait for it to fully start, then plug in the router. Wait 2 minutes, then test. - Contact your home internet service provider (ISP) if the issue persists. Summit IT cannot troubleshoot your home ISP connection, but can advise on using a mobile hotspot as a backup.

10. Are you trying to access a resource that requires VPN (an internal tool, file server, or internal website)?

If yes → Ensure GlobalProtect is connected. Open the GlobalProtect icon in your system tray (Windows) or menu bar (Mac), and verify the status shows "Connected." If not connected, click Connect and authenticate with your Okta credentials. Approve the Okta Verify push notification.

If VPN connects but the resource is still unreachable → The issue may be with the resource itself rather than your network. Check with colleagues to see if others can access it. If no one can, submit a P2 ticket at https://help.summitit.com.

11. Is the Summit-Corp SSID visible in the Wi-Fi list but you cannot connect?

This often indicates an authentication failure with the 802.1X network (Okta credentials rejected or certificate issue).

  • Forget the Summit-Corp network and reconnect, re-entering your Okta username and password carefully.
  • Ensure your Okta password has not expired. If it has, reset it first at https://summit.okta.com using mobile data or a different device.
  • If you still cannot join, submit a ticket — the Service Desk may need to re-push the Wi-Fi profile via Intune or Jamf.

Quick Checklist

Before calling the Service Desk, confirm you have tried:

  • [ ] Verified Wi-Fi is enabled on your laptop
  • [ ] Confirmed you are connected to Summit-Corp (office) or your home network (remote)
  • [ ] Tested a basic website (google.com) to confirm general internet
  • [ ] Flushed DNS and renewed IP address
  • [ ] Restarted your laptop's Wi-Fi adapter
  • [ ] Checked whether other devices on the same network are affected
  • [ ] Restarted home router and modem if working remotely (wait 2 min after restart)
  • [ ] Verified GlobalProtect VPN is connected if trying to access internal tools
